# 08 - Future Research

## Overview

Chronicle / Legacy Protocol is currently a concept and architecture research project. Future work should focus on clearer specifications, risk analysis, governance implications, and prototype design before any implementation code is introduced.

## Research Areas

| Area | Research Need |
|---|---|
| Contribution verification | Define evidence standards and attestation strength |
| Reputation design | Prevent transferability, farming, and over-simplified scoring |
| Archive architecture | Determine what belongs on-chain, off-chain, hashed, or referenced |
| Privacy | Protect contributor identity and disclosure preferences |
| Governance | Study how memory and reputation can inform decisions without capturing them |
| AI mentoring | Define source-linked, uncertainty-aware AI assistance |
| Knowledge inheritance | Preserve contributor experience, decisions, and operational lessons |
| Interoperability | Explore portable contribution and archive formats |

## Open Questions

- How should disputes and corrections be handled?
- Can contribution history remain useful without becoming exclusionary?
- How can communities avoid turning reputation into status hierarchy?
- What evidence is strong enough for protocol-level memory?
- How can AI summarize ecosystem history without distorting it?
- How should private, sensitive, or security-related contributions be preserved?

## Next Milestones

1. Expand the manifesto into a full principles document.
2. Create a formal architecture diagram.
3. Define an attestation schema.
4. Draft a contributor reputation model.
5. Design a privacy and disclosure framework.
6. Develop a prototype data model without smart contract code.
7. Compare Chronicle with existing identity, attestation, and reputation systems.

## Design Position

Future research should protect the project from premature implementation. The concept should become clear, falsifiable, and reviewable before code is written.
